# Вариант 18
Лабораторные работы по ОСям (IPC, семафоры и РОП, UNIX domain sockets)

---

## Лабораторная работа 3

### Клиент
1. Создать очередь сообщений.
2. Определить в текущем каталоге файл, модифицированный последним.
3. Передать в очередь сообщений имя этого файла.

### Сервер
1. Прочитать из очереди последнее сообщение.
2. Определить:
   - количество строк в указанном файле;
   - общее число сообщений в очереди.
3. Вывести результаты в стандартный поток вывода.
4. Удалить очередь сообщений.

---

## Лабораторная работа 4

### Сервер
1. Создать разделяемую область памяти (РОП) и набор семафоров (НС).
2. Записать в РОП:
   - идентификаторы всех процессов пользователя;
   - приоритеты процессов;
   - суммарное рабочее время процессов;
   - имена терминалов, связанных с процессами.
3. Печатать полученные от клиентов сообщения.
4. В конце работы удалить РОП и НС.

### Клиент 1
1. Извлечь из РОП PID процесса-«долгожителя».
2. Определить текущее состояние процесса.

### Клиент 2
1. Извлечь из РОП PID процесса с наименьшим приоритетом.
2. Определить количество семафоров в наборе.

---

## Лабораторная работа 5

### Сервер
1. Создать UNIX domain socket типа *virtual circuit*.
2. Назначить ему адрес.
3. Из всех сообщений клиентов выбрать имена файлов, которые присутствуют в обоих списках.

### Клиент 1
1. Создать UNIX domain socket типа *virtual circuit*.
2. Отправить сообщение со списком файлов текущего каталога размером не более 100 байт.

### Клиент 2
1. Создать UNIX domain socket типа *virtual circuit*.
2. Отправить сообщение со списком файлов текущего каталога, созданных за последние три дня.
